## 1. 최초 설정

### Supabase 프로젝트 설정

시스템을 처음 사용하려면 Supabase에 데이터베이스를 구성해야 합니다.

1. [Supabase](https://supabase.com) 계정 생성 및 새 프로젝트 생성
2. **SQL Editor** 에서 `database_v2.1_full_schema.sql` 전체 내용을 붙여넣고 실행
3. **Authentication → Providers → Email** 에서 `Confirm email` 옵션 OFF (개발/소규모 운영 권장)
4. **Database → Replication** 에서 `game_info`, `scores` 테이블 실시간 활성화 확인

## 2. 회원가입 및 로그인

### 회원가입

1. 사이트 접속 → 로그인 화면
2. **"사용자 등록"** 버튼 클릭
3. 다음 정보 입력:
   - **이메일**: 로그인에 사용할 이메일
   - **비밀번호**: 6자 이상
   - **비밀번호 확인**
   - **나만의 코드**: 스코어보드 고정 URL에 사용되는 고유 식별자

#### 코드 규칙
| 규칙 | 예시 |
|------|------|
| 소문자 영문으로 시작 | ✅ `dan`, ❌ `1dan` |
| 영문 소문자, 숫자, 하이픈만 허용 | ✅ `my-team`, ❌ `My Team` |
| 3~20자 | ✅ `abc`, ❌ `ab` |

> 코드는 등록 후 변경이 어려우므로 신중하게 입력하세요.
> 이미 사용 중인 코드는 등록이 불가합니다.

### 로그인

1. 이메일과 비밀번호 입력
2. **로그인** 버튼 클릭 → 관리자 패널(`/a`)로 이동

### 비밀번호 재설정

1. 로그인 화면 → **"패스워드 재설정"** 버튼
2. 이메일 입력 후 **"패스워드 재설정"** 버튼 클릭
3. 이메일로 발송된 링크를 통해 재설정

## 3. 경기 등록

### 새 경기 만들기

1. 관리자 패널(`/a`) → 우측 상단 **"+ New Game"** 버튼 클릭
2. 경기 정보 입력:

| 항목 | 설명 |
|------|------|
| 경기 타이틀 | 스코어보드 상단에 표시되는 경기명 (예: `2025 전국대회 결승`) |
| 경기 일시 | 날짜 및 시간 선택 |
| 경기장 | 경기 장소 |
| 원정팀 | 팀명 입력 또는 저장된 팀 선택 |
| 홈팀 | 팀명 입력 또는 저장된 팀 선택 |

> 이전에 등록한 팀명을 입력하면 콤보박스에서 선택 가능 → 로고, 배경색 자동 적용

3. **"경기 등록"** 버튼 클릭
4. 새 경기는 기본적으로 **비활성(is_live=false)** 상태로 생성됩니다.

### 경기 수정

경기 카드 우측 상단 **⋮** 메뉴 → **"✏️ 경기 수정"** 클릭

### 경기 삭제

경기 카드 우측 상단 **⋮** 메뉴 → **"🗑️ 삭제"** 클릭 → 확인

## 4. 경기 활성화 (is_live)

고정 URL(`/oa/코드`)은 현재 **활성화된 경기(is_live=true)** 를 자동으로 표시합니다.

### 경기 활성화 방법

경기 카드 하단 **"▶ 활성화"** 버튼 클릭

- 클릭 즉시 해당 경기만 `is_live=true`, 나머지는 모두 `is_live=false`
- 버튼이 **"🔴 방송중"** 으로 변경
- OBS에 연결된 스코어보드가 WebSocket을 통해 자동으로 해당 경기로 전환

### 경기 전환 흐름

```
[▶ 활성화] 클릭
    ↓
DB 업데이트 (is_live 변경)
    ↓
WebSocket으로 실시간 이벤트 전달
    ↓
스코어보드 자동 전환 (새로고침 불필요)
```

## 5. 스코어 컨트롤러 사용법

### 컨트롤러 열기

경기 카드 → **"🎮 스코어보드 컨트롤러"** 버튼 클릭 (새 창으로 열림)

또는 직접 URL 입력:
- `https://your-domain/c/{gameId}` — 특정 경기 ID로 접속
- `https://your-domain/c/{코드}` — 활성 경기 자동 연결

### 기본 조작

#### 점수 조작
| 버튼 | 동작 |
|------|------|
| 팀 점수 `+` | 해당 팀 점수 1 증가 |
| 팀 점수 `-` | 해당 팀 점수 1 감소 (0 이하 불가) |

#### 이닝 조작
| 버튼 | 동작 |
|------|------|
| `-` | 이전 이닝(초/말)으로 |
| `+` | 다음 이닝(초/말)으로 |

> 주의) 이닝 전환 시 BSO(볼, 스트라이크, 아웃)와 주루 상황이 자동 초기화됩니다.

#### BSO 조작
| 항목 | 범위 | 설명 |
|------|------|------|
| B (볼) | 0~3 | `Ball` 버튼으로 조작 |
| S (스트라이크) | 0~2 | `Strike` 버튼으로 조작 |
| O (아웃) | 0~2 | `Out` 버튼으로 조작 |

#### 주루 상황
- 1루(◆), 2루(◆), 3루(◆) 버튼을 클릭하여 주자 ON/OFF 토글

#### 홈/원정 팀 전환
경기 카드 → **⋮** 메뉴 → **"🔄 홈/원정 전환"**

## 6. 스코어보드 오버레이 설정

### 스코어보드 종류

| 타입 | 크기 | URL | 특징 |
|------|------|-----|------|
| A타입 | 330×180px | `/oa/{gameId 또는 코드}` | 세로형, 팀·점수·이닝·BSO·주루 표시 |
| B타입 | 1200×47px | `/ob/{gameId 또는 코드}` | 가로형, 콤팩트한 하단 바 스타일 |

### OBS 브라우저 소스 추가

1. OBS → **소스 추가** → **브라우저**
2. **URL** 입력:
   - 고정 URL (권장): `https://your-domain/oa/your-code`
   - 특정 경기: `https://your-domain/oa/12345`
3. **너비/높이** 설정:
   - A타입: 너비 `330`, 높이 `180`
   - B타입: 너비 `1920`, 높이 `47`
4. **사용자 지정 CSS** (선택):
   ```css
   body { background-color: rgba(0,0,0,0); }
   ```
5. **확인** 클릭

> 고정 URL(`/oa/코드`)을 사용하면 경기가 바뀌어도 URL을 수정할 필요가 없습니다.

### URL 복사하기

**AdminPanel 상단** → 내 고정 URL 섹션에서 📋 버튼 클릭으로 클립보드 복사

또는 경기 카드 → **⋮** 메뉴 → **"🔗 A타입 URL 복사"** / **"🔗 B타입 URL 복사"**

## 7. 고정 URL 사용법

### 개념

사용자 코드를 기반으로 URL을 고정하고, `is_live=true` 인 경기를 자동으로 표시하는 기능입니다.

```
https://your-domain/oa/dan
    → "dan" 코드 유저의 is_live=true 경기를 ScoreboardA로 표시
```

### 고정 URL 목록

| URL | 설명 |
|-----|------|
| `/oa/{코드}` | A타입 스코어보드 (활성 경기 자동) |
| `/ob/{코드}` | B타입 스코어보드 (활성 경기 자동) |
| `/c/{코드}` | 스코어 컨트롤러 (활성 경기 자동) |

### 실제 사용 시나리오

```
1. overlay에 /oa/dan URL 한 번 설정 (이후 변경 불필요)

2. 경기 시작 전
   → AdminPanel에서 해당 경기 [▶ 활성화] 클릭

3. 경기 중
   → 스코어보드가 해당 경기 데이터를 실시간 표시

4. 다음 경기
   → 다음 경기 카드에서 [▶ 활성화] 클릭
   → 스코어보드가 자동으로 새 경기로 전환
```

### 내 고정 URL 확인

AdminPanel(`/a`) 상단에 **"내 스코어보드 고정 URL"** 섹션에서 확인 및 복사 가능

## 8. 테마 커스터마이징

### 설정 모달 열기

경기 카드 → **"🎨 테마 변경"** 버튼 클릭

### 설정 항목

#### 팀 로고
- 홈팀/원정팀 로고 이미지 URL 입력
- 입력 후 미리보기에서 즉시 확인 가능
- 팀 로고 참조: [대한야구소프트볼협회](http://www.korea-baseball.com/info/team/team_list)

> 로고 이미지는 공개적으로 접근 가능한 URL이어야 합니다.
> 권장 형식: PNG (투명 배경), 500KB 이하

#### 폰트 크기
| 항목 | 범위 | 설명 |
|------|------|------|
| 시합 타이틀 | 16~50px | 스코어보드 상단 경기명 크기 |
| 팀명 | 20~60px | 팀 이름 표시 크기 |

슬라이더 또는 숫자 입력으로 조정하며, 미리보기에서 실시간 확인 가능

#### 배경색
- 홈팀/원정팀 각각 배경색 지정
- 색상 피커 또는 HEX 코드 직접 입력
- 자주 사용하는 팀 색상 미리 지정된 팔레트 제공

### 저장

**"저장"** 버튼 클릭 → 즉시 스코어보드에 반영

> 저장 시 팀 정보(로고, 배경색)가 teams 테이블에 자동 저장되어,
> 다음 경기 등록 시 콤보박스에서 바로 불러올 수 있습니다.

## 9. 선수 관리

### 선수 관리 화면 접속

관리자 패널 → **"👤 선수 관리"** 버튼 또는 `/p` 직접 접속

### 팀 선택 및 선수 등록

1. 상단 팀 목록에서 팀 선택
2. **"+ 선수 등록"** 버튼 클릭
3. 선수 정보 입력:

| 항목 | 필수 | 설명 |
|------|------|------|
| 이름 | ✅ | 선수 이름 |
| 등번호 | | 등번호 (숫자) |
| 주포지션 | | 투수, 포수, 내야수 등 |
| 부포지션 | | 추가 포지션 |
| 투타 | | 우투우타, 좌투좌타 등 |
| 구분 | | 투수(P) / 타자(B) |

## 10. 선수 프로필 팝업

경기 중 특정 선수의 프로필을 스코어보드 화면에 슬라이드 애니메이션으로 표시하는 기능입니다.

### 팝업 표시 방법

스코어 컨트롤러(`/c/{gameId}`) 하단 **"👤 선수 프로필 팝업"** 섹션:

1. **팀 선택**: 어웨이 / 홈 토글
2. **투수/타자선택**: 투수/타자 선택하여 선수 선택
3. **선수 선택**: 드롭다운에서 선수 선택 (등번호 + 이름 표시)
4. **표시 위치**: 왼쪽 중앙 / 오른쪽 중앙
5. **표시 시간**: 초 단위 입력
6. **"확인"** 버튼 클릭 → 스코어보드에 즉시 표시

### 팝업 표시 내용

- 선수 사진 (세로형: 225×345px / 가로형: 375×270px 자동 감지)
- 등번호, 이름
- 주포지션 · 부포지션 · 투타

## 11. 자주 묻는 질문

### Q. 스코어보드가 업데이트되지 않아요
- 브라우저 오른쪽 하단에 빨간 점이 표시되면 WebSocket 연결이 끊긴 상태입니다.
- 페이지를 새로고침하면 자동으로 재연결됩니다.
- Supabase 대시보드에서 `game_info`, `scores` 테이블의 Realtime이 활성화되어 있는지 확인하세요.

### Q. /oa/코드 접속 시 아무것도 표시되지 않아요
- 해당 코드의 유저가 존재하는지 확인하세요.
- AdminPanel에서 경기를 **[▶ 활성화]** 클릭하여 `is_live=true` 경기를 지정했는지 확인하세요.

### Q. 회원가입 시 "email rate limit exceeded" 오류가 발생해요
- Supabase 무료 플랜의 이메일 발송 제한(시간당 3~4건)에 걸린 것입니다.
- **Authentication → Providers → Email** 에서 `Confirm email` 을 OFF로 설정하세요.

### Q. 코드를 잘못 입력했어요
- Supabase 대시보드 → **Table Editor** → `profiles` 테이블에서 직접 수정 가능합니다.

### Q. 기존에 가입된 유저에게 코드를 부여하고 싶어요
- Supabase SQL Editor에서 직접 INSERT합니다:
  ```sql
  INSERT INTO profiles (id, code)
  VALUES ('유저-uuid', '원하는코드');
  ```
- 유저 UUID는 **Authentication → Users** 에서 확인할 수 있습니다.

### Q. 로고 이미지가 표시되지 않아요
1. 이미지 URL이 `https://` 로 시작하는지 확인
2. 해당 URL에 직접 브라우저로 접속하여 이미지가 표시되는지 확인
3. CORS 정책으로 차단된 경우 Supabase Storage에 이미지를 업로드하여 사용

### Q. 화면에서 스코어보드 배경이 검은색으로 나와요
- OBS 브라우저 소스의 **사용자 지정 CSS** 에 다음을 추가하세요:
  ```css
  body { background-color: rgba(0,0,0,0); }
  ```
